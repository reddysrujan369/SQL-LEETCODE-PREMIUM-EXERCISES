# CEO Subordinate Hierarchy — MySQL Recursive CTE

## Problem Summary
Find all subordinates of the CEO (direct + indirect), their hierarchy level,
and salary difference from the CEO.

---

## Final Query

```sql
WITH RECURSIVE
    T AS (
        SELECT
            employee_id,
            employee_name,
            0               AS hierarchy_level,
            manager_id,
            salary
        FROM Employees
        WHERE manager_id IS NULL

        UNION ALL

        SELECT
            E.employee_id,
            E.employee_name,
            T.hierarchy_level + 1 AS hierarchy_level,
            E.manager_id,
            E.salary
        FROM T
        JOIN Employees E ON T.employee_id = E.manager_id
    ),
    P AS (
        SELECT salary
        FROM Employees
        WHERE manager_id IS NULL
    )
SELECT
    T.employee_id   AS subordinate_id,
    T.employee_name AS subordinate_name,
    T.hierarchy_level,
    T.salary - P.salary AS salary_difference
FROM T
CROSS JOIN P
WHERE T.hierarchy_level != 0
ORDER BY 3, 1;
```

---

## Flow of Execution

### Step 1 — CTE T Anchor (seed)
- Scans `Employees` for `manager_id IS NULL`
- Finds **Alice** → inserted into T with `hierarchy_level = 0`
- ---

### Step 2 — CTE T Recursive Iteration 1
- JOIN condition: `T.employee_id = E.manager_id`
- Alice's `employee_id = 1` matches `manager_id = 1`
- New rows: Bob, Charlie with `level = 0 + 1 = 1`
- ---

### Step 3 — CTE T Recursive Iteration 2
- Seeds: Bob (id=2), Charlie (id=3)
- Matches: David, Eve (manager_id=2) and Frank, Grace (manager_id=3)
- All get `level = 1 + 1 = 2`
- ---

### Step 4 — CTE T Recursive Iteration 3
- Seeds: David(4), Eve(5), Frank(6), Grace(7)
- Only Eve (id=5) has a report → Helen (manager_id=5)
- Helen gets `level = 2 + 1 = 3`
- ---

### Step 5 — Recursion Terminates
- Helen (id=8) has no direct reports
- Next iteration produces 0 rows → MySQL stops
- ---

### Step 6 — CTE P
- Single scan for `manager_id IS NULL`
- Returns 1 row: `{ salary: 150000 }`

---

### Step 7 — CROSS JOIN T × P
- P has exactly 1 row → safe cross join
- CEO salary (150000) is attached to every row in T
- 8 × 1 = 8 rows

---

### Step 8 — WHERE + ORDER BY
- `WHERE T.hierarchy_level != 0` → drops Alice → 7 rows remain
- `ORDER BY 3, 1` → sort by hierarchy_level ASC, employee_id ASC

---

### Final Output

| subordinate_id | subordinate_name | hierarchy_level | salary_difference |
|----------------|------------------|-----------------|-------------------|
| 2              | Bob              | 1               | -30000            |
| 3              | Charlie          | 1               | -40000            |
| 4              | David            | 2               | -45000            |
| 5              | Eve              | 2               | -50000            |
| 6              | Frank            | 2               | -55000            |
| 7              | Grace            | 2               | -52000            |
| 8              | Helen            | 3               | -60000            |

---

## Time Complexity

| Component            | Complexity     | Reason                                              |
|----------------------|----------------|-----------------------------------------------------|
| Anchor (CTE T)       | O(N)           | Full scan of Employees for manager_id IS NULL       |
| Each recursive step  | O(N)           | JOIN scans Employees once per iteration             |
| All recursive steps  | O(D × N)       | D iterations × O(N) each                           |
| CTE P                | O(N)           | Single scan for manager_id IS NULL                  |
| CROSS JOIN T × P     | O(N)           | P is always 1 row → N × 1                          |
| WHERE filter         | O(N)           | Single pass to drop level=0 row                     |
| ORDER BY             | O(N log N)     | Sort by hierarchy_level, employee_id                |
| **Overall**          | **O(D × N)**   | Recursion dominates; sort is secondary              |

---

## Space Complexity

| Component   | Complexity | Reason                                          |
|-------------|------------|-------------------------------------------------|
| CTE T       | O(N)       | Stores all N employees, each exactly once       |
| CTE P       | O(1)       | Always exactly 1 row                            |
| **Overall** | **O(N)**   | Linear in number of employees                   |

---

## Worst vs Best Case

| Scenario                        | Depth D | Time Complexity |
|---------------------------------|---------|-----------------|
| Pure chain (1→2→3→...→N)        | N       | O(N²)           |
| Flat org (CEO → everyone)       | 1       | O(N)            |
| This dataset (N=8, D=3)         | 3       | effectively O(N)|

---

## Key Concepts

- **Recursive CTE** terminates when the recursive part produces 0 new rows
- **CROSS JOIN with P** is safe only because exactly 1 CEO exists
- **hierarchy_level = 0** is used internally to mark the CEO and is excluded in the final WHERE
- **salary_difference is always negative** here since all subordinates earn less than Alice
